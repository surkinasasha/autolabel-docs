# Функциональные требования

## UC‑01: Создание проекта

**Акторы:** авторизованный пользователь, система  
**Предусловия:** пользователь авторизован и находится в рабочем пространстве  

### Основной поток
1. Пользователь нажимает «Создать проект».
2. Система отображает форму: название (3–50 симв.), описание (≤500), тип задачи, список классов.
3. Пользователь заполняет поля и добавляет хотя бы один класс.
4. Система проверяет:
   - название не пустое, не из пробелов, укладывается в лимит;
   - описание (если есть) ≤ 500 символов;
   - выбран тип задачи;
   - название уникально в рабочем пространстве;
   - не превышен лимит проектов (3 для бесплатной версии);
   - список классов не пуст, названия уникальны.
5. При успешной проверке система создаёт проект, сохраняет в БД и открывает его страницу.

### Альтернативный поток A1 (отмена)
- Пользователь нажимает «Отмена» или закрывает форму.
- Система закрывает форму, возврат в рабочее пространство.

### Исключительные потоки
- **E1.** Название не заполнено или состоит из пробелов.  
  Система подсвечивает поле и сообщает: «Название проекта является обязательным для заполнения».
- **E2.** Название вне лимитов (менее 3 или более 50 символов).  
  Сообщение: «Название проекта должно содержать от 3 до 50 символов».
- **E3.** Описание > 500 символов.  
  Сообщение: «Описание проекта не должно превышать 500 символов».
- **E4.** Тип задачи не выбран.  
  Сообщение: «Необходимо выбрать тип задачи».
- **E5.** Проект с таким названием уже существует.  
  Сообщение: «Проект с таким названием уже существует в рабочем пространстве».
- **E6.** Достигнут лимит проектов (3 бесплатных).  
  Сообщение: «Лимит по проектам достигнут. В бесплатной версии можно создать не более 3 проектов».
- **E7.** Системная ошибка при создании.  
  Сообщение: «Не удалось создать проект из-за внутренней ошибки сервера. Попробуйте позже».
- **E8.** Список классов пуст.  
  Сообщение: «Необходимо добавить хотя бы один класс для разметки».
- **E9.** Названия классов дублируются или некорректны.  
  Сообщение: «Названия классов должны быть уникальными и содержать от 1 до 30 символов».

### Бизнес-правила
- Название проекта уникально в пределах рабочего пространства.
- Тип задачи нельзя изменить после создания.
- Бесплатный тариф – до 3 проектов, платный – без ограничений.

## UC‑02: Импорт данных

**Акторы:** авторизованный пользователь, система  
**Предусловия:** пользователь авторизован и открыл проект  

### Основной поток
1. Пользователь нажимает «Загрузить данные».
2. Система открывает область загрузки.
3. Пользователь передаёт файлы (одиночные, папкой, drag‑and‑drop).
4. Система проверяет:
   - формат (jpg, png, bmp, tiff, jpeg, webp);
   - размер файла ≤ 50 МБ;
   - файл не повреждён;
   - общее количество с учётом уже загруженных не превышает лимит тарифа (2000 для бесплатной версии).
5. При успешной проверке начинается загрузка с отображением прогресса.
6. Система уведомляет об успешной загрузке, помечает файлы как «неразмеченные».

### Альтернативный поток A1 (отмена)
- Пользователь закрывает диалог выбора файлов.
- Система закрывает диалог, файлы не загружаются.

### Исключительные потоки
- **E1.** Неподдерживаемый формат. Система отфильтровывает такие файлы и предупреждает.
- **E2.** Размер файла > 50 МБ. Сообщение для каждого такого файла: «Файл [имя] слишком велик (макс. 50 МБ)».
- **E3.** Выбрана пустая папка. Сообщение: «В выбранном источнике не найдены подходящие изображения».
- **E4.** Системная ошибка при загрузке. Сообщение: «Произошла ошибка при загрузке файлов. Повторите позже».
- **E5.** Превышен лимит количества файлов для тарифа. Сообщение: «Превышен лимит количества файлов для текущего тарифа (максимум 2000 файлов)».

### Бизнес-правила
- Структура вложенных папок не сохраняется – все изображения добавляются в общий датасет.
- Бесплатным пользователям доступно до 2 000 изображений.

## UC‑03: Ручная разметка (детекция)

**Акторы:** авторизованный пользователь, система  
**Предусловия:** открыт проект с загруженными изображениями, тип задачи – детекция  

### Основной поток
1. Пользователь выбирает изображение из галереи.
2. Система открывает редактор разметки (панель классов, инструменты, масштаб, отмена/повтор).
3. Пользователь выбирает класс.
4. Пользователь создаёт Bounding Box (мышью): зажимает левую кнопку – тянет – отпускает.
5. Система отображает прямоугольник с классом.
6. Пользователь может корректировать (перемещать, изменять размер).
7. Шаги 3–6 повторяются для всех объектов.
8. Пользователь нажимает «Сохранить» или переходит к другому изображению (автосохранение).
9. Система сохраняет аннотации и обновляет статус изображения на «размечено».
10. Пользователь может перейти к следующему изображению.

### Альтернативный поток A1 (пропуск)
- Пользователь нажимает «Пропустить».
- Система помечает изображение как «неразмеченное» и возвращает к галерее.

### Исключительный поток E1 (ошибка сохранения)
- Сообщение: «Не удалось сохранить разметку. Попробуйте еще раз».
- Аннотация не сохраняется, пользователь остаётся в редакторе.

## UC‑04: Ручная разметка (сегментация)

**Акторы:** авторизованный пользователь, система  
**Предусловия:** открыт проект с загруженными изображениями, тип задачи – сегментация  

### Основной поток
1. Пользователь выбирает изображение из галереи.
2. Система открывает редактор разметки (классы, кисть, ластик, заливка, масштаб, отмена).
3. Пользователь выбирает класс.
4. Пользователь создаёт маску: рисование кистью, лассо, заливка или умная сегментация по клику.
5. Система отображает маску.
6. Пользователь может корректировать маску.
7. Шаги 3–6 повторяются для всех объектов.
8. Пользователь нажимает «Сохранить» (или автосохранение при переходе).
9. Система сохраняет маски и обновляет статус изображения на «размечено».
10. Пользователь переходит к следующему изображению.

### Альтернативный поток A1 (пропуск)
- Пользователь нажимает «Пропустить».
- Система помечает изображение как «неразмеченное» и возвращает к галерее.

### Исключительный поток E1 (ошибка сохранения)
- Сообщение: «Не удалось сохранить разметку. Попробуйте еще раз».
- Аннотация не сохраняется, пользователь остаётся в редакторе.

## UC‑05: Пакетная автоматическая разметка

**Акторы:** авторизованный пользователь, система  
**Предусловия:** открыт проект с загруженными изображениями, у пользователя активна подписка или доступны вычислительные лимиты  

### Основной поток
1. Пользователь переходит в раздел «Авторазметка».
2. Система отображает настройки:
   - выбор области обработки (все, неразмеченные, выбранные);
   - информация о лимитах (осталось N изображений);
   - поле для текстового промпта;
   - кнопка «Запустить авторазметку».
3. Система проверяет:
   - выбрано хотя бы одно изображение;
   - промпт заполнен;
   - количество не превышает лимита.
4. Система запускает автоматическую детекцию или сегментацию в зависимости от типа задачи.
5. Система уведомляет о завершении, показывает результаты, сортирует по уверенности и подсвечивает изображения с уверенностью &lt;50%.

### Альтернативный поток A1 (отмена)
- Пользователь нажимает «Отмена» до запуска.
- Система возвращается на страницу проекта без изменений.

### Исключительные потоки
- **E1.** Изображения не выбраны. Сообщение: «Необходимо выбрать изображения».
- **E2.** Превышен лимит изображений. Сообщение: «Выбрано X изображений, но ваш лимит составляет Y. Пожалуйста, уменьшите выборку или перейдите на платный тариф».
- **E3.** Промпт не заполнен. Сообщение: «Необходимо ввести текстовый промпт».
- **E4.** GPU‑серверы недоступны. Сообщение: «Сервис авторазметки временно недоступен. Попробуйте позже». Пользователь может оставить задачу в очереди или отменить её.

### Бизнес-правила
- Для бесплатного тарифа — максимум 500 изображений для авторазметки в месяц.

## UC‑06: Экспорт размеченного датасета

**Акторы:** авторизованный пользователь, система  
**Предусловия:** открыт проект, есть хотя бы одно размеченное изображение  

### Основной поток
1. Пользователь нажимает «Экспорт».
2. Система открывает окно настроек:
   - выбор формата аннотаций (COCO JSON, YOLO, Pascal VOC; для сегментации только COCO);
   - информация о лимите выгрузок (осталось X из Y);
   - выбор изображений (все / только размеченные);
   - кнопка «Экспортировать».
3. Пользователь выбирает формат и фильтр, нажимает «Экспортировать».
4. Система проверяет:
   - выбрано хотя бы одно изображение;
   - лимит выгрузок не превышен.
5. Система формирует ZIP‑архив: изображения + аннотации + data.yaml.
6. Система запускает скачивание архива и сообщает об успехе.

### Альтернативный поток A1 (отмена)
- Пользователь нажимает «Отмена» до запуска экспорта.
- Система закрывает окно, лимиты не списываются.

### Исключительные потоки
- **E1.** Нет размеченных изображений. Сообщение: «Нет размеченных изображений для экспорта».
- **E2.** Лимит выгрузок исчерпан. Сообщение: «Месячный лимит экспорта исчерпан (5 из 5). Обновление лимита произойдет 1-го числа следующего месяца или при переходе на платный тариф».
- **E3.** Ошибка архивации. Сообщение: «Ошибка формирования архива. Пожалуйста, попробуйте еще раз позже». Лимит не списывается.

### Бизнес-правила
- Бесплатный тариф – не более 5 выгрузок датасета в месяц.

## UC‑07: Выбор и оплата тарифного плана

**Акторы:** авторизованный пользователь, система, внешняя платёжная система  
**Предусловия:** пользователь авторизован  

### Основной поток
1. Пользователь переходит в раздел «Тарифы».
2. Система отображает тарифы («Бесплатный», «Профессиональный», «Корпоративный»), их стоимость и ограничения.
3. Пользователь выбирает платный тариф и нажимает «Оформить подписку».
4. Система перенаправляет на страницу оплаты.
5. Пользователь вводит платёжные данные и подтверждает.
6. Платёжная система обрабатывает транзакцию и возвращает статус.
7. Система получает подтверждение оплаты.
8. Система активирует тариф: обновляет лимиты, фиксирует дату окончания подписки.
9. Система сообщает об успешной активации.

### Альтернативный поток A1 (отмена автопродления)
- Пользователь в настройках профиля нажимает «Отменить подписку».
- Система запрашивает подтверждение.
- После подтверждения статус подписки меняется на «Отменена, активна до [дата]». Лимиты сохраняются до окончания оплаченного периода.

### Исключительный поток E1 (ошибка оплаты)
- Платёжная система возвращает ошибку.
- Система отображает сообщение об ошибке. Подписка не активируется. Пользователь остаётся на странице оплаты для повторной попытки.

### Бизнес-правила
- **Даунгрейд:** при окончании платной подписки и превышении лимитов (например, >3 проектов) создание новых проектов и загрузка файлов блокируется до приведения к лимитам.
- При неоплате система пытается списать средства несколько раз, затем понижает тариф до Бесплатного.

## UC‑08: Управление ролями

**Акторы:** авторизованный пользователь (создатель проекта), приглашённый пользователь, система  
**Предусловия:** открыта страница проекта  

### Основной поток
1. Пользователь открывает настройки проекта → «Участники».
2. Система показывает список участников с ролями (Администратор, Разметчик, Наблюдатель).
3. Пользователь нажимает «Пригласить участника».
4. Система открывает форму: поле email и выбор роли.
5. Пользователь вводит email и выбирает роль.
6. Система проверяет:
   - корректность email;
   - не превышен лимит участников для тарифа;
   - пользователь ещё не участник проекта и нет активного приглашения.
7. Если проверки пройдены, система создаёт приглашение и отправляет письмо со ссылкой.
8. Приглашённый пользователь переходит по ссылке, авторизуется / регистрируется.
9. Система добавляет его в проект с выбранной ролью.
10. Администратор может изменить роль участника (выпадающий список → подтверждение).
11. Администратор может удалить участника (кнопка «Исключить» → подтверждение). Участник теряет доступ.

### Альтернативный поток A1 (самостоятельный выход)
- Участник (не владелец) может покинуть проект через настройки.
- Система удаляет его из списка участников.

### Исключительные потоки
- **E1.** Неверный формат email. Сообщение: «Введите корректный email‑адрес».
- **E2.** Превышение лимита участников. Сообщение: «Достигнут максимальный лимит участников для вашего тарифа. Чтобы пригласить больше участников, перейдите на другой тарифный план».
- **E3.** Пользователь уже участник проекта. Сообщение: «Пользователь с таким email уже является участником проекта или имеет активное приглашение».
- **E4.** Ошибка отправки email. Приглашение сохраняется в статусе «Ошибка отправки», владелец может повторить отправку позже.

### Бизнес-правила
- Создатель проекта автоматически получает роль Администратора.
- Администратор может приглашать, исключать участников, менять роли, а также управлять настройками и экспортом.
- Разметчик может загружать, размечать (вручную и автоматически) изображения.
- Наблюдатель может только просматривать данные и аннотации.
