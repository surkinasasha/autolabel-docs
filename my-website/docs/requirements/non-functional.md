# Нефункциональные требования

| ID | Категория | Название | Описание | Приоритет | Обоснование |
|----|-----------|----------|----------|-----------|-------------|
| NFR-001 | Производительность | Скорость работы умной сегментации по клику | Время отклика при генерации маски объекта (SAM) после клика — не более 1 с | Must have | Критично для удобства работы и сокращения времени разметки. Ориентир – инференс на GPU NVIDIA T4. |
| NFR-002 | Производительность | Скорость автоматической детекции и сегментации | Обработка одного изображения (GroundingDINO/SAM) в фоновом режиме — не более 3 с | Must have | Пользователи ожидают существенной экономии времени. Ориентир – инференс на GPU T4. |
| NFR-003 | Производительность | Скорость импорта датасета | Загрузка и первичная обработка — не менее 100 изображений/мин (размер до 5 МБ). Отклик UI при выборе файлов — до 2 с | Should have | Быстрая загрузка улучшает UX, особенно на больших датасетах. Основано на анализе конкурентов. |
| NFR-004 | Производительность | Скорость формирования архива для экспорта | Формирование ZIP‑архива (до 1000 изображений ≤ 5 МБ) — не более 3 мин | Should have | Длительное ожидание снижает эффективность работы. |
| NFR-005 | Безопасность | Безопасность хранения паролей | Хранение паролей в виде хэша | Must have | Защита учётных данных пользователей. |
| NFR-006 | Совместимость | Кросс-браузерность | Корректная работа в двух последних версиях Chrome, Yandex Browser, Firefox | Should have | Поддержка широкого круга клиентов. |
| NFR-007 | Надёжность | Uptime | 99.0 % в первой версии (допустимо до 7.2 ч downtime в месяц) | Should have | На этапе MVP возможна нестабильность, в будущем показатель будет повышен. |
| NFR-008 | Удобство использования | Интуитивный интерфейс | 80 % новых пользователей выполняют цикл «Загрузка → Разметка → Экспорт» без обращения в поддержку | Should have | Заявленное конкурентное преимущество перед open‑source решениями. |
| NFR-009 | Нормативно-правовые | Локализация в РФ | Серверы БД и файловые хранилища размещены на территории РФ (152‑ФЗ) | Must have | Ключевое требование для госзаказчиков и крупных российских клиентов. |
| NFR-010 | Наблюдаемость | Логирование и мониторинг | Логирование всех ключевых операций и ошибок. Хранение логов — не менее 30 дней в JSON | Should have | Необходимо для оперативного выявления сбоев и соблюдения uptime. |
| NFR-011 | Наблюдаемость | Метрики для бизнес-анализа | Сбор анонимизированных метрик (проекты, изображения, авторазметка, экспорты) | Could have | Важно для отслеживания продуктовых и бизнес-показателей. |
| NFR-012 | Ёмкость | Лимиты для бесплатной версии | Максимальный размер одного изображения — 50 МБ, максимум файлов — 2000 | Should have | Предотвращает переполнение дискового пространства и неконтролируемые расходы. |
| NFR-013 | Ёмкость | Общий объём хранилища данных | Поддержка не менее 100 ТБ на старте с возможностью бесшовного расширения до 1 ПБ и более | Must have | Расчёт на 1000 активных пользователей в MVP. |
| NFR-014 | Ёмкость | Одновременные пользователи | Поддержка до 500 одновременных активных сессий | Must have | Заложено с запасом (~20% от ожидаемых 1000 пользователей). |
| NFR-015 | Масштабируемость | Горизонтальное масштабирование авторазметки | Архитектура на основе очереди задач, возможность добавления GPU‑воркеров без остановки сервиса | Must have | Обеспечивает устойчивость при росте нагрузки и возможность быстрого расширения. |
| NFR-016 | Безопасность | Управление ролями | Разграничение прав: Администратор, Разметчик, Наблюдатель | Should have | Защита от случайного/намеренного удаления данных в корпоративных командах. |
| NFR-017 | Безопасность | Конфиденциальность данных | Доступ к файлам только по временным ссылкам (срок жизни < 15 мин) | Must have | Исключает массовое скачивание датасетов при перехвате ссылок. |
| NFR-018 | Ёмкость | Одновременная авторазметка | Поддержка до 50 одновременных сессий авторазметки (одна сессия – до 1000 изображений) | Must have | Рассчитано исходя из ожидаемого количества пользователей в MVP. |
