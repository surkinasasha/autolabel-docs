## Описание сущностей

### 1. Пользователь, Подписка и Проект

- **Атрибуты:** ID пользователя, email, хеш пароля, тип подписки, остаток лимитов, ID проекта, название проекта, список классов.
- **Акторы:** Пользователи (управление профилем и проектами), биллинговая система (обновление подписок).
- **Характер взаимодействия:** OLTP – высокие требования к консистентности из-за управления доступом и платными подписками.

### 2. Изображение (бинарный файл)

- **Атрибуты:** Бинарный файл изображения.
- **Акторы:** Пользователь (загрузка/скачивание), GPU-кластер, Frontend (отображение в браузере).
- **Характер взаимодействия:** Запись один раз, чтение много раз. Транзакции не требуются.

### 3. Метаданные изображения

- **Атрибуты:** ID изображения, ID проекта, URL ссылки на бинарный файл, статус разметки (unlabeled, auto_labeled, labeled), разрешение.
- **Акторы:** Backend, пользователи (поиск и фильтрация).
- **Характер взаимодействия:** OLTP – изменение статусов.

### 4. Аннотация (Разметка)

- **Атрибуты:** ID аннотации, ID изображения, ID класса, тип (Bounding Box, Polygon), координаты, уверенность модели, флаг low-confidence.
- **Акторы:** ML-модели, пользователь (ручное редактирование, удаление, добавление).
- **Характер взаимодействия:** OLTP – частая запись и обновление координат в процессе работы в редакторе.

### 5. ML-задача (Очередь обработки)

- **Атрибуты:** ID задачи, ID пользователя, тип модели, параметры (промпт), текущий статус (pending, processing, completed, failed), прогресс (%), ссылка на результат, timestamp создания и завершения.
- **Акторы:** Backend (создание и обновление статуса), GPU-кластер (обновление прогресса и статуса), пользователь (просмотр списка задач).
- **Характер взаимодействия:** OLTP – важна консистентность статусов для отображения актуального состояния.
